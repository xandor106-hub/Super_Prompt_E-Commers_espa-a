⚙️ ESPECIFICACIÓN GLOBAL DE EFICIENCIA (APLICABLE A TODOS LOS PROMPTS)
================================================================================

[Especificación de Eficiencia y Estructura]

## Principios de Eficiencia Obligatorios

1. **Colocación (Colocation):** Los archivos deben vivir lo más cerca posible de donde se usan. Si un componente solo se usa en una página, va dentro de la carpeta de esa página, no en `/shared`.

2. **Barrel Exports Controlados:** Usar `index.ts` solo en la raíz de cada feature para exponer su API pública. No abusar de barrels que rompan el tree-shaking.

3. **Lazy Loading por Defecto:** Los componentes de páginas o features grandes deben cargarse con `next/dynamic` o `React.lazy`. Especificar en la estructura qué componentes son candidatos a lazy loading.

4. **Zero Dependencias Innecesarias:** Si se sugiere alguna librería npm, justificar por qué es necesaria y si tiene alternativa nativa más ligera.

5. **TypeScript Estricto:** Todo archivo debe ser `.ts` o `.tsx`. No se acepta JavaScript plano.

6. **Tailwind CSS para Estilos:** Asumir Tailwind como capa de estilos. No generar archivos `.css` o `.module.css` separados a menos que sea estrictamente necesario.

7. **Server Components por Defecto:** En Next.js App Router, todo componente debe ser Server Component a menos que necesite interactividad (`'use client'`). Marcar explícitamente en la estructura cuáles son cliente.

8. **Carga Progresiva de Imágenes:** Todas las imágenes deben usar `next/image` con `priority={false}` excepto las primeras visibles (LCP) que llevarán `priority`.

## Formato de Salida Esperado (para prompts que generan código)

1. **Árbol de Carpetas Completo** con responsabilidad de cada archivo.
2. **Lista de Decisiones de Eficiencia:** Breve párrafo explicando por qué se estructuró así.
3. **Código** de los archivos solicitados.

[Reglas Obligatorias para el Código Generado]

## Regla 1: Variables Centralizadas
- Todos los colores deben definirse en variables CSS dentro de `:root` en `globals.css`.
- Los componentes NUNCA deben contener valores hex, rgb, px o rem quemados.

## Regla 2: Componentes Sin Valores Concretos
- Los componentes usan nombres semánticos: `primary`, `secondary`, `danger`, `success`.
- Ejemplo CORRECTO: `<button className="bg-primary text-primary-foreground">`
- Ejemplo INCORRECTO: `<button className="bg-[#2a2a2a] text-white">`

## Regla 3: Componentes Base (UI Primitives)
- Crear componentes base como `<Button>`, `<Card>`, `<Input>` en `shared/ui/components/`.
- Los componentes de features DEBEN usar estos primitivos, NUNCA estilos directos.

## Regla 4: Cero Estilos Inline
- PROHIBIDO usar `style={{}}` en componentes React.
- PROHIBIDO usar clases arbitrarias de Tailwind como `bg-[#xxx]` o `w-[123px]`.
